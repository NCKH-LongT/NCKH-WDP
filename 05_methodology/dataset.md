# Dataset

## 1. Current Dataset Status

The project currently does not have official historical SEAL Hackathon source code submissions and judge scores. Therefore, the initial evaluation will use a prototype dataset.

## 2. Prototype Dataset Plan

| Item | Description |
|---|---|
| Problem statement | A sample hackathon-style programming task based on SEAL themes |
| Rubric | Dynamic rubric adapted from Fall 2025 or Spring 2026 criteria |
| Source code submissions | 10–20 sample submissions or GitHub repositories |
| Human judge scores | Scores assigned by 2–3 human evaluators |
| AI scores | Scores generated by the LLM-based AI Evaluation Service |
| Judge overrides | Records of accept/override actions |
| Timing data | Manual grading time and AI-assisted grading time |

## 3. Possible Data Sources

1. Existing student projects.
2. Team-created sample submissions with different quality levels.
3. Previous programming assignments.
4. Simplified hackathon projects created for testing.
5. Public toy repositories if licensing allows reuse.

## 4. Dataset Fields

Each sample should contain:

```text
submissionId
teamId
problemStatement
rubricId
sourceCodeSnapshot or commitDiff
README excerpt
humanJudgeScores
aiSuggestedScores
judgeOverrideRecords
gradingTime
```

## 5. Missing Information

- Official SEAL Hackathon source code submissions are not yet available.
- Real judge scores are not yet available.
- The final number of submissions for evaluation is not confirmed.
- The final LLM API/model is not confirmed.

## 6. Dataset Limitation

Because the initial dataset may be small and partially simulated, the result should be presented as a prototype evaluation rather than a large-scale benchmark.
